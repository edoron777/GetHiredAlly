CONTEXT:
This is a multi-step implementation to replace the current CV Optimizer 
dual-path interface (Easy Way + Deep Dive) with a single unified 
"Document View" interface inspired by Microsoft Word.

PROJECT OVERVIEW:
- Goal: Create unified results page with Word-like document display
- Files affected: pages/cv-optimizer/, components/cv-optimizer/
- Dependencies: Existing ActionBar, DocStyler, database tables (unchanged)

WHAT WE'RE BUILDING:
1. New ResultsPage with tabbed interface (Document View / List View)
2. Word-like CV display with gray background and white "paper"
3. Inline issue markers (colored underlines) on CV text
4. Info icons (ⓘ) that open tip modals
5. Left sidebar with score and issue navigation
6. Content selector for exports (CV / Recommendations / Both)
7. Quick Format Tools panel for cosmetic fixes

WHAT WE'RE KEEPING (NO CHANGES):
- Upload Screen
- Scanning Screen (only change redirect destination)
- Database tables (cv_analyses, cv_analysis_issues, cv_issue_types)
- ActionBar component
- DocStyler component

WHAT WE'RE REMOVING (after new interface is ready):
- CrossroadsPage
- EasyWayPage
- Dual-path selection UI
